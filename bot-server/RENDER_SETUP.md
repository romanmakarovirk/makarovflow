# Настройка бота на Render.com

## Важно! После деплоя на Render нужно добавить переменную окружения

1. Откройте ваш сервис на Render.com
2. Перейдите в раздел **Environment**
3. Добавьте новую переменную:
   - **Key**: `WEBHOOK_URL`
   - **Value**: URL вашего сервиса (например: `https://mindflow-bot-5hph.onrender.com`)

4. Нажмите **Save Changes**
5. Render автоматически перезапустит бот

## Проверка работы

После перезапуска:
1. Откройте логи бота на Render
2. Должна быть строка: `✅ Webhook установлен: https://your-url.onrender.com/webhook`
3. Откройте бота в Telegram
4. Отправьте команду `/start`
5. Бот должен ответить приветственным сообщением с кнопкой

## Если бот не отвечает

Проверьте:
- ✅ Переменная `TELEGRAM_BOT_TOKEN` установлена
- ✅ Переменная `WEBHOOK_URL` установлена и совпадает с URL сервиса
- ✅ В логах нет ошибок
- ✅ Сервис работает (зеленый статус на Render)

## Технические детали

Бот теперь использует **webhook режим** вместо long polling:
- Telegram отправляет обновления на `https://your-url.onrender.com/webhook`
- Это работает лучше на serverless платформах типа Render
- Бот всегда готов отвечать, даже после периода неактивности
