# ВВЕДЕНИЕ

В современном мире цифровых технологий все больше людей стремятся к саморазвитию и самопознанию. Одним из эффективных инструментов для этого является ведение личного дневника, который помогает систематизировать мысли, анализировать эмоции и отслеживать прогресс в достижении целей.

## Актуальность проекта

По данным исследований, регулярное ведение дневника способствует улучшению психологического состояния, помогает справляться со стрессом и развивает навыки рефлексии. Однако традиционные бумажные дневники имеют ряд недостатков: их легко потерять, сложно систематизировать записи, невозможно быстро найти нужную информацию.

Существующие цифровые решения для ведения дневников часто требуют установки отдельных приложений, регистрации на различных платформах и не всегда удобны в использовании. Большинство популярных приложений являются платными или содержат навязчивую рекламу.

В связи с этим возникла потребность в создании простого, удобного и доступного инструмента для ведения личного дневника, который был бы интегрирован в уже знакомую пользователям среду — мессенджер Telegram.

## Цель проекта

Разработать веб-приложение MakarovFlow для ведения личного дневника с функциями самоанализа, интегрированное в Telegram в виде Mini App, доступное для использования на любых устройствах.

## Задачи проекта

Для достижения поставленной цели необходимо решить следующие задачи:

1. Изучить существующие решения для ведения электронных дневников и выявить их преимущества и недостатки
2. Изучить современные технологии веб-разработки (React, TypeScript, Vite)
3. Освоить работу с Telegram Mini Apps API
4. Спроектировать архитектуру приложения и базу данных
5. Разработать пользовательский интерфейс приложения
6. Реализовать основной функционал: создание записей, категоризация, поиск
7. Внедрить систему анализа настроения и статистики
8. Протестировать приложение на различных устройствах
9. Развернуть приложение на хостинге и интегрировать с Telegram

## Объект и предмет исследования

**Объект исследования:** процесс разработки веб-приложений для личного использования.

**Предмет исследования:** методы и технологии создания интерактивных веб-приложений, интегрированных в экосистему Telegram.

## Методы исследования

- Анализ существующих решений и технологий
- Проектирование и моделирование
- Программирование и разработка
- Тестирование и отладка
- Опытная эксплуатация

## Практическая значимость

Разработанное приложение может использоваться для:
- Ведения личного дневника и журнала мыслей
- Отслеживания эмоционального состояния
- Анализа личного прогресса и достижений
- Развития навыков рефлексии и самоанализа

Приложение является полностью бесплатным, работает без подключения к интернету (после первой загрузки) и не требует регистрации, так как использует аутентификацию Telegram.

# ГЛАВА 1. ТЕОРЕТИЧЕСКИЕ ОСНОВЫ РАЗРАБОТКИ WEB-ПРИЛОЖЕНИЙ

## 1.1 Анализ существующих решений для ведения дневников

Для понимания требований к разрабатываемому приложению я провел анализ существующих решений для ведения электронных дневников.

### Популярные приложения для ведения дневников

**Day One** — одно из самых популярных приложений для ведения дневника на iOS и macOS. Преимущества: красивый интерфейс, синхронизация между устройствами, поддержка фото и видео. Недостатки: платная подписка (от $34.99/год), доступно только для устройств Apple.

**Journey** — кроссплатформенное приложение с функциями геолокации и анализа настроения. Преимущества: работает на всех платформах, поддержка облачной синхронизации. Недостатки: требует регистрации, бесплатная версия ограничена.

**Notion** — универсальный инструмент для заметок, который можно использовать как дневник. Преимущества: гибкость настройки, множество функций. Недостатки: сложный для новичков, избыточный функционал для простого дневника.

**Диаро** — приложение для Android с простым интерфейсом. Преимущества: бесплатное, поддержка экспорта данных. Недостатки: устаревший дизайн, нет версии для других платформ.

### Выявленные недостатки существующих решений

1. **Платность** — большинство качественных приложений требуют подписки
2. **Платформозависимость** — многие приложения доступны только для определенных ОС
3. **Необходимость регистрации** — требуется создание отдельного аккаунта
4. **Сложность** — избыточный функционал усложняет использование
5. **Отсутствие интеграции** — приложения существуют отдельно от привычных инструментов

### Преимущества разрабатываемого решения

Исходя из анализа, я определил ключевые преимущества проекта MakarovFlow:

- **Бесплатность** — полностью бесплатное приложение без рекламы
- **Кроссплатформенность** — работает на любом устройстве через браузер
- **Интеграция с Telegram** — не требует отдельной регистрации
- **Простота** — минималистичный интерфейс, сфокусированный на главном
- **Офлайн-режим** — возможность работы без интернета
- **Приватность** — данные хранятся локально на устройстве пользователя

## 1.2 Обзор современных технологий веб-разработки

Для разработки современного веб-приложения необходимо использовать актуальные технологии, которые обеспечивают высокую производительность, удобство разработки и хорошую поддержку.

### React — библиотека для создания пользовательских интерфейсов

**React** — это JavaScript-библиотека, разработанная компанией Meta (Facebook), предназначенная для создания интерактивных пользовательских интерфейсов.

Основные преимущества React:
- **Компонентный подход** — интерфейс разбивается на независимые переиспользуемые компоненты
- **Virtual DOM** — обеспечивает высокую производительность за счет оптимизации обновлений
- **Большое сообщество** — множество готовых библиотек и решений
- **React Hooks** — современный подход к управлению состоянием компонентов

В моем проекте React используется для создания всех элементов интерфейса: форм для записей, списков дневниковых записей, экранов статистики и настроек.

### TypeScript — типизированный JavaScript

**TypeScript** — это язык программирования, разработанный Microsoft, который является надмножеством JavaScript и добавляет статическую типизацию.

Преимущества использования TypeScript:
- **Проверка типов** — ошибки выявляются на этапе разработки, а не в runtime
- **Автодополнение** — улучшенная поддержка в редакторах кода
- **Рефакторинг** — безопасное переименование и изменение кода
- **Документация** — типы служат документацией к коду

Применение в проекте: TypeScript помог избежать множества ошибок при работе со сложными структурами данных, такими как записи дневника, настройки пользователя и состояние приложения.

### Vite — инструмент сборки

**Vite** — современный инструмент для сборки фронтенд-приложений, созданный автором Vue.js.

Преимущества Vite:
- **Быстрый запуск** — мгновенный старт dev-сервера
- **Hot Module Replacement** — изменения отображаются без перезагрузки страницы
- **Оптимизированная сборка** — использует Rollup для production
- **Поддержка TypeScript** — работает из коробки

В проекте Vite используется для разработки и сборки финальной версии приложения.

### Дополнительные технологии

**Tailwind CSS** — utility-first CSS-фреймворк для быстрой разработки интерфейсов. Позволяет создавать стили прямо в JSX-разметке, что ускоряет разработку.

**Zustand** — легковесная библиотека для управления состоянием приложения. Проще и меньше Redux, но достаточно мощная для проекта такого масштаба.

**Dexie.js** — обертка над IndexedDB для работы с локальной базой данных в браузере. Позволяет хранить записи дневника локально на устройстве пользователя.

**Framer Motion** — библиотека для создания анимаций. Делает интерфейс более живым и приятным в использовании.

**React Router** — библиотека для маршрутизации (навигации между страницами) в React-приложениях.

## 1.3 Telegram Mini Apps как платформа для приложений

### Что такое Telegram Mini Apps

**Telegram Mini Apps** (ранее Web Apps) — это платформа для запуска веб-приложений внутри мессенджера Telegram. Mini Apps открываются в встроенном браузере и имеют доступ к специальному API для интеграции с Telegram.

### Преимущества Telegram Mini Apps

1. **Встроенная аутентификация** — приложение автоматически получает данные пользователя Telegram (ID, имя, фото профиля)
2. **Большая аудитория** — Telegram используют более 900 миллионов человек по всему миру
3. **Нет необходимости в установке** — приложение открывается прямо в мессенджере
4. **Кроссплатформенность** — работает на всех устройствах, где есть Telegram
5. **Интеграция с ботами** — можно создать бота для взаимодействия с приложением
6. **Безопасность** — данные передаются через защищенное соединение

### Telegram Web Apps API

API предоставляет следующие возможности:

- **Информация о пользователе** — получение ID, имени, языка интерфейса
- **Управление интерфейсом** — изменение цветов, темы, кнопок
- **Главная кнопка** — настраиваемая кнопка внизу приложения
- **Кнопка "Назад"** — обработка нажатия системной кнопки
- **Закрытие приложения** — контролируемое закрытие
- **Haptic feedback** — тактильная обратная связь
- **Облачное хранилище** — небольшое хранилище для настроек (до 1024 ключей)

### Интеграция в проекте MakarovFlow

В проекте я использовал Telegram Mini Apps для:

1. **Аутентификации** — вместо создания системы регистрации используется Telegram ID
2. **Адаптации интерфейса** — приложение автоматически подстраивается под тему Telegram (светлая/темная)
3. **Навигации** — кнопка "Назад" работает естественно для пользователей
4. **Улучшения UX** — haptic feedback при важных действиях
5. **Синхронизации настроек** — базовые настройки сохраняются в Cloud Storage

### Создание базы данных

Я создал файл `src/db/database.ts` с настройкой Dexie.js для работы с IndexedDB:

```typescript
import Dexie, { Table } from 'dexie';

export interface Entry {
  id: string;
  date: Date;
  content: string;
  mood: 'happy' | 'sad' | 'neutral' | 'anxious' | 'excited';
  tags: string[];
  favorite: boolean;
}

class MakarovFlowDB extends Dexie {
  entries!: Table<Entry>;

  constructor() {
    super('MakarovFlowDB');
    this.version(1).stores({
      entries: 'id, date, mood, favorite'
    });
  }
}

export const db = new MakarovFlowDB();
```

Это позволяет приложению работать полностью офлайн, храня все данные локально.

### Управление состоянием приложения

С помощью Zustand я создал централизованное хранилище состояния в файле `src/store/useStore.ts`:

```typescript
import { create } from 'zustand';

interface AppState {
  entries: Entry[];
  loading: boolean;
  addEntry: (entry: Entry) => Promise<void>;
  deleteEntry: (id: string) => Promise<void>;
  updateEntry: (id: string, updates: Partial<Entry>) => Promise<void>;
  loadEntries: () => Promise<void>;
}

export const useStore = create<AppState>((set) => ({
  entries: [],
  loading: false,

  addEntry: async (entry) => {
    await db.entries.add(entry);
    set(state => ({ entries: [...state.entries, entry] }));
  },

  deleteEntry: async (id) => {
    await db.entries.delete(id);
    set(state => ({ entries: state.entries.filter(e => e.id !== id) }));
```
