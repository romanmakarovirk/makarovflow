## ПРОБЛЕМАТИКА И АКТУАЛЬНОСТЬ

### Выявленная проблема
В современном мире школьники и студенты сталкиваются с высокими требованиями к организации времени и управлению стрессом. Существующие приложения для трекинга настроения и управления задачами имеют следующие недостатки:
1. Сложность использования - требуют длительного обучения
2. Отсутствие приватности - данные хранятся на внешних серверах
3. Необходимость регистрации и установки отдельных приложений
4. Отсутствие комплексного подхода - либо только настроение, либо только задачи
5. Платность многих решений

### Статистические данные
- По данным ВОЗ, более 70% подростков испытывают стресс из-за учебной нагрузки
- 62% учащихся имеют проблемы с организацией времени и выполнением задач
- 78% респондентов (опрос учащихся 9-11 классов, 2024) отметили сложности с отслеживанием своего настроения

### Целевая аудитория
- **Основная**: Школьники и студенты (14-25 лет)
- **Дополнительная**: Педагоги, родители, молодые специалисты
- **Характеристики**: Активные пользователи Telegram, заинтересованные в самоорганизации и личностном росте, ценящие приватность данных

## ФУНКЦИОНАЛЬНОСТЬ ПРИЛОЖЕНИЯ (ПОДРОБНО)

### 1. ДНЕВНИК НАСТРОЕНИЯ (Journal)

**Основные возможности:**
- Ежедневные записи с отслеживанием:
  * Настроение (шкала 1-10)
  * Эмоциональное состояние (эмодзи)
  * Уровень энергии (1-10)
  * Качество сна (часы и оценка 1-10)
  * Текстовые заметки

**Компоненты:**
- **MultiStepCheckIn** — многошаговая форма для добавления записи (4 шага)
- **JournalCalendar** — календарь с историей записей и цветовой индикацией настроения
- **Компактные карточки аналитики:**
  * Средний балл настроения за неделю
  * Средний уровень энергии
  * Среднее качество сна
  * Текущий стрик (дни подряд ведения дневника)

**Виджеты для учебы:**
- **Расписание занятий** (ScheduleTab) — добавление уроков по дням недели
- **Домашние задания** (HomeworkTab) — трекер домашки с приоритетами
- **Калькуляторы** (CalculatorsTab):
  * GPA Calculator (средний балл по системе 4.0)
  * Калькулятор среднего балла (российская система 2-5)
  * Калькулятор процента выполнения

**AI-инсайты:**
- Автоматический анализ записей
- Выявление корреляций между настроением, сном и энергией
- Персонализированные рекомендации

### 2. УМНЫЕ ЗАДАЧИ (Tasks)

**Дизайн-философия:**
Минималистичный дизайн в стиле приложения Things 3

**Основные возможности:**
- Создание задач с названием и заметками
- Установка дедлайнов (дата и время)
- Настройка напоминаний
- Категоризация по спискам
- Отметка выполнения с анимацией

**Система фильтров (4 типа):**
1. **Все 📋** — все активные задачи
2. **Сегодня ⭐** — задачи со статусом "today"
3. **Скоро 📅** — задачи с установленными дедлайнами
4. **Готово ✅** — завершенные задачи

**Особенности UI/UX:**
- Анимированные чекбоксы с spring physics эффектом
- Красная граница для просроченных задач
- Полноценная модальная форма для создания задач
- Плавные переходы между фильтрами
- Haptic feedback (вибрация при действиях)

### 3. AI-ПОМОЩНИК (AIAssistant)

**Технология:**
Keyword-based интеллектуальная система (без внешних API для экономии)

**Возможности:**
- Контекстное понимание пользовательских данных
- Анализ истории записей в дневнике
- Анализ выполнения задач
- Персональные рекомендации по улучшению продуктивности
- Suggested questions на основе собранных данных

**Компактные метрики в шапке чата:**
- Средний балл настроения
- Текущий стрик
- Количество активных задач
- Процент выполненных задач

**Лимиты использования:**
- Бесплатный план: 10 запросов в день
- Premium план: 50 запросов в день

**Примеры вопросов, на которые отвечает AI:**
- "Когда я наиболее продуктивен?"
- "Как сон влияет на мое настроение?"
- "Какие задачи я чаще всего откладываю?"
- "Советы по улучшению настроения"

### 4. НАСТРОЙКИ (Settings)

**Разделы:**

**Язык:**
- Переключение между русским и английским
- Мгновенное применение без перезагрузки
- Сохранение выбора в локальном хранилище

**Данные:**
- Экспорт всех данных в JSON формат
- Возможность очистки всех данных
- Информация об использовании хранилища

**Premium блок:**
- Информация о Premium возможностях

**О приложении:**

## ТЕХНОЛОГИЧЕСКИЙ СТЕК (ДЕТАЛЬНО)

### Frontend (Telegram Mini App)

**Основной фреймворк:**
- **React 18.3** — современная версия с Concurrent Features
- **Vite 5.0** — сверхбыстрый build tool с HMR (Hot Module Replacement)
  * Время холодного старта: < 1 секунда
  * Время HMR: < 50ms

**Стилизация:**
- **TailwindCSS 3.4** — utility-first CSS фреймворк
  * JIT (Just-In-Time) режим для минимального размера bundle
  * Кастомная цветовая палитра
  * Responsive дизайн из коробки
  * Градиенты: purple/blue/cyan (AI), pink (настроение), yellow (энергия), blue (сон)

**Анимации:**
- **Framer Motion 11.0** — production-ready animation library
  * Spring physics для естественных анимаций
  * Layout animations для smooth transitions
  * Gesture animations (tap, hover, drag)
  * Достижение 60 FPS на всех устройствах

**База данных:**
- **Dexie.js 4.0** — обертка над IndexedDB
  * Версия схемы: 3
  * 7 таблиц с индексами
  * Асинхронные операции
  * Миграции между версиями схемы
  * Полная типизация через JSDoc

**State Management:**
- **Zustand 4.5** — минималистичный state manager
  * Без boilerplate кода
  * React hooks based API
  * Middleware для persist (localStorage)
  * Размер: ~1KB

**Интернационализация:**
- **i18next 23.0** + react-i18next
  * 2 языка: русский (по умолчанию), английский
  * Lazy loading переводов
  * Pluralization и interpolation
  * Сохранение выбранного языка

**Иконки:**
- **Lucide React 0.400** — современные SVG иконки
  * Tree-shakeable (только используемые иконки в bundle)
  * Размер каждой иконки: ~500 байт
  * Легко кастомизируются через props

**Telegram интеграция:**
- **@twa-dev/sdk 7.0** — Telegram Web Apps SDK
  * Haptic feedback
  * Theme colors
  * Viewport управление
  * BackButton navigation
  * MainButton для CTA

### Backend (Telegram Bot)

**Runtime:**
- **Node.js 20.x** — LTS версия с актуальными фичами

**Bot Framework:**
- **Grammy 1.30** — современный TypeScript-first фреймворк
  * Webhook режим (для Render.com)
  * Автоматическая обработка команд
  * Inline keyboards поддержка
  * Error handling middleware

**Функциональность бота:**
- Обработка команды `/start`
- Предоставление Web App URL через inline keyboard
- Webhook endpoint для получения обновлений от Telegram
- Health check endpoint для мониторинга

### Инфраструктура и деплой

**Frontend хостинг:**
- **Vercel** (ранее Netlify)
  * Автоматический деплой из GitHub (main branch)
  * Edge Network для быстрой загрузки
  * HTTPS из коробки
  * Custom domain поддержка
  * Build time: ~1 минута

**Backend хостинг:**
- **Render.com** (Free tier)
  * Автоматический деплой из GitHub
  * Webhook режим для Telegram
  * Environment variables для токенов
  * Автоматический перезапуск при ошибках
  * Регион: Frankfurt (Europa)

**Version Control & CI/CD:**
- **GitHub**
  * Основная ветка: main
  * Feature branches для разработки
  * GitHub Actions для автотестов (planned)
  * Branch protection rules

**Мониторинг:**
- **UptimeRobot** (optional)
  * Проверка доступности бота каждые 5 минут
  * Email уведомления при downtime
  * 99.9% uptime guarantee

### База данных (IndexedDB, версия 3)

**Таблица: journal_entries** (записи дневника)
```javascript
{
  id: number (auto-increment),
  date: string (ISO 8601),
  mood: number (1-10),
  moodEmoji: string,
  energy: number (1-10),
  sleepHours: number (0-24),
  sleepQuality: number (1-10),
  notes: string (optional),
  tags: array<string> (optional),
  createdAt: timestamp,
  updatedAt: timestamp
}
// Индексы: date, mood, energy, createdAt
```

**Таблица: tasks** (задачи)
```javascript
{
  id: number (auto-increment),
  title: string (required),
  notes: string (optional),
  list: string (enum: 'inbox', 'today', 'upcoming'),
  when: string (enum: 'inbox', 'today', 'evening', 'someday'),
  deadline: timestamp (optional),
  reminder: string (time, optional),
  completed: boolean,
  createdAt: timestamp,
  completedAt: timestamp (nullable)

// Индексы: list, when, deadline, completed, createdAt
```

### Ключевые функции и хелперы

**database.js — основные методы:**
- `journalEntries.getLastN(n)` — получить последние N записей
- `journalEntries.getByDateRange(start, end)` — записи за период
- `journalEntries.getStats()` — статистика (средние значения)
- `tasks.create(data)` — создать задачу
- `tasks.getByFilter(filter)` — получить задачи по фильтру
- `tasks.toggleComplete(id)` — переключить статус выполнения
- `userStats.updateEntryStats()` — обновить статистику записей
- `userStats.updateStreak()` — обновить стрик
- `settings.getLanguage()` — получить текущий язык
- `settings.setLanguage(lang)` — установить язык

**analytics.js — расчетные функции:**
- `calculateAverageMood(entries)` — средний балл настроения
- `calculateAverageEnergy(entries)` — средняя энергия
- `calculateAverageSleep(entries)` — средний сон
- `calculateStreak(entries)` — текущий стрик
- `getMoodTrend(entries)` — тренд настроения (↑ ↓ →)
- `getProductivityScore(tasks, entries)` — оценка продуктивности
- `getCorrelations(entries)` — корреляции между метриками

**chatService.js — AI логика:**
- `analyzeUserData(entries, tasks, stats)` — анализ данных пользователя
- `generateResponse(message, context)` — генерация ответа
- `getSuggestedQuestions(context)` — рекомендуемые вопросы
- `detectIntent(message)` — определение намерения пользователя

### Решенные проблемы и вызовы

**1. Бот засыпал на Render Free Tier**
- **Проблема:** Render.com "усыпляет" бесплатные сервисы после 15 мин неактивности
- **Решение:**
  * Переключение с long polling на webhook режим
  * Автоматическая установка webhook при старте
  * Health check endpoint для мониторинга

**2. IndexedDB сложность**
- **Проблема:** Низкоуровневое API IndexedDB неудобно для работы
- **Решение:**
  * Использование Dexie.js обертки
  * Создание хелпер-функций для частых операций
  * Proper миграции между версиями схемы

**3. Анимации тормозили на слабых устройствах**
- **Проблема:** 60 FPS не достигался на старых Android
- **Решение:**
  * Использование CSS transforms вместо layout properties
  * GPU acceleration (will-change, transform3d)
  * Throttling сложных анимаций
  * Conditional rendering для тяжелых компонентов

**4. Telegram theme colors конфликтовали с дизайном**
- **Проблема:** Автоматические цвета темы Telegram ломали визуал
- **Решение:**
  * Отключение автоматических цветов
  * Использование собственной темной палитры
  * Optional адаптация под Telegram (planned)
