## 1. ОБЩАЯ ИНФОРМАЦИЯ О ПРОЕКТЕ

### Название и назначение
**MakarovFlow** — веб-приложение для управления личной продуктивностью и мониторинга психоэмоционального состояния в формате Telegram Mini App.

### Основная идея
Создать комплексное решение, которое помогает учащимся:
- Отслеживать свое эмоциональное состояние (настроение, энергия, сон)
- Управлять задачами и дедлайнами
- Анализировать паттерны продуктивности с помощью AI
- Организовывать учебный процесс (расписание, домашка, оценки)

**Ключевое отличие**: Полная приватность данных — все хранится локально на устройстве пользователя, никакой передачи на внешние серверы.

### Целевая аудитория
- **Основная**: Школьники и студенты (14-25 лет)
- **Дополнительная**: Молодые специалисты, педагоги, родители
- **Характеристики**: Активные пользователи Telegram, заинтересованные в самоорганизации, ценящие приватность данных

## 4. ФУНКЦИОНАЛЬНОСТЬ (ПОДРОБНО)

### 📖 Дневник настроения (Journal)

**Что записывается:**
- **Настроение** — оценка от 1 до 10 + эмодзи
- **Энергия** — уровень от 1 до 10
- **Сон** — количество часов (0-24) + качество сна (1-10)
- **Заметки** — текстовые записи о дне
- **Теги** — категории событий (необязательно)

**Как работает:**
1. Пользователь нажимает "Добавить запись"
2. Открывается многошаговая форма (4 шага):
   - Шаг 1: Выбор настроения (слайдер + эмодзи)
   - Шаг 2: Уровень энергии (слайдер)
   - Шаг 3: Качество сна (часы + оценка)
   - Шаг 4: Заметки (текстовое поле)
3. Данные сохраняются в IndexedDB
4. Отображаются в календаре с цветовой индикацией

**Компоненты:**
- `MultiStepCheckIn` — форма добавления записи
- `JournalCalendar` — календарь с историей
- Компактные карточки аналитики:
  - Средний балл настроения за неделю
  - Средний уровень энергии
  - Среднее качество сна
  - Текущий стрик (дни подряд)

**Виджеты для учебы:**
- **Расписание** — добавление уроков по дням недели (предмет, время, учитель, кабинет)
- **Домашка** — трекер домашних заданий с дедлайнами и приоритетами
- **Калькуляторы**:
  - GPA Calculator (средний балл по системе 4.0)
  - Российский средний балл (2-5)
  - Процент выполнения

### ✅ Менеджер задач (Tasks)

**Дизайн-философия:**
Минималистичный дизайн в стиле приложения Things 3 (популярный iOS task manager)

**Возможности:**
- Создание задач с названием и заметками
- Установка дедлайнов (дата + время)
- Настройка напоминаний
- Категоризация по спискам (inbox, today, upcoming)
- Отметка выполнения с анимацией

**Система фильтров (4 типа):**
1. **📋 Все** — все активные задачи
2. **⭐ Сегодня** — задачи со статусом "today"
3. **📅 Скоро** — задачи с дедлайнами
4. **✅ Готово** — завершенные задачи

**Особенности UI:**
- Анимированные чекбоксы (spring physics)
- Красная граница для просроченных задач
- Haptic feedback при отметке
- Плавные переходы между фильтрами

### 🤖 Аналитика (AI Assistant)

**Какие данные анализируются:**
- История записей в дневнике (настроение, энергия, сон)
- Статистика выполнения задач
- Текущий стрик
- Корреляции между метриками

**Как работает алгоритм:**
1. **Keyword-based система** (без внешних API):
   - Анализ ключевых слов в вопросе пользователя
   - Сопоставление с паттернами (regex)
2. **Получение контекста** из базы данных:
   - Последние 7-30 дней записей
   - Статистика задач
   - Тренды метрик
3. **Генерация ответа** на основе шаблонов

**Выявляемые паттерны:**
- Корреляция между сном и настроением
  - Пример: "Когда вы спите 7-8 часов, настроение на 15% выше"
- Оптимальное время для продуктивности
  - Пример: "Вы наиболее продуктивны с 10:00 до 14:00"
- Тренды настроения (↑ улучшается, ↓ ухудшается, → стабильно)
- Процент выполнения задач
- Частота просроченных задач

**Визуализация:**
- Компактные карточки метрик в шапке чата
- Suggested questions на основе данных
- Текстовые ответы с эмодзи

**Лимиты:**
- Free: 10 запросов в день
- Premium: 50 запросов в день (planned)

**Примеры вопросов:**
- "Когда я наиболее продуктивен?"
- "Как сон влияет на мое настроение?"
- "Какие задачи я чаще откладываю?"
- "Советы по улучшению настроения"

### 🎓 Инструменты для учёбы

**1. Расписание (Schedule):**

## 5. КЛЮЧЕВЫЕ ТЕХНИЧЕСКИЕ РЕШЕНИЯ

### Почему Telegram Mini Apps?

**Преимущества:**
1. **Не требует установки** — открывается в 1 клик из Telegram
2. **Кроссплатформенность** — один код работает везде
3. **Большая аудитория** — 800+ млн пользователей Telegram
4. **Встроенные фичи** — haptic feedback, theme, backbutton
5. **Безопасность** — HTTPS, sandbox окружение
6. **Простота деплоя** — обычное веб-приложение + бот

**Недостатки (и как решены):**
- Нет синхронизации между устройствами → будущая фича (Telegram Cloud Storage)
- Ограничения localStorage → используем IndexedDB (неограниченно)
- Зависимость от Telegram → но это и фича (привычная среда)

### Локальное хранение данных (IndexedDB)

**Почему IndexedDB, а не LocalStorage:**
- **Объем**: LocalStorage ≤ 10MB, IndexedDB ≥ 50MB (неограниченно с разрешением)
- **Типы данных**: LocalStorage — только строки, IndexedDB — любые объекты
- **Скорость**: IndexedDB быстрее для больших объемов
- **Индексы**: IndexedDB поддерживает индексы для быстрого поиска
- **Транзакции**: IndexedDB поддерживает ACID транзакции

**Как реализовано:**
1. **Dexie.js обертка** — упрощает работу с IndexedDB
2. **Версионирование схемы** — текущая версия 3
3. **7 таблиц**:
   - `journal_entries` — записи дневника
   - `tasks` — задачи
   - `schedule` — расписание
   - `homework` — домашка
   - `ai_messages` — история AI чата
   - `user_stats` — статистика
   - `settings` — настройки
4. **Индексы** для быстрого поиска:
   - По дате (для записей)
   - По дедлайну (для задач)
   - По дню недели (для расписания)

**Хелперы для работы:**
```javascript
// database.js
journalEntries.getLastN(7) // Последние 7 записей
tasks.getByFilter('today') // Задачи на сегодня
userStats.updateStreak()   // Обновить стрик
settings.setLanguage('en') // Сменить язык
```

### Алгоритм аналитики и корреляций

**Шаг 1: Сбор данных**
```javascript
const entries = await journalEntries.getByDateRange(startDate, endDate);
```

**Шаг 2: Расчет средних значений**
```javascript
const avgMood = entries.reduce((sum, e) => sum + e.mood, 0) / entries.length;
const avgEnergy = entries.reduce((sum, e) => sum + e.energy, 0) / entries.length;
const avgSleep = entries.reduce((sum, e) => sum + e.sleepHours, 0) / entries.length;
```

**Шаг 3: Выявление корреляций**
```javascript
// Группировка по количеству сна
const sleepGroups = {
  '<6h': entries.filter(e => e.sleepHours < 6),
  '6-8h': entries.filter(e => e.sleepHours >= 6 && e.sleepHours <= 8),
  '>8h': entries.filter(e => e.sleepHours > 8)
};

// Расчет среднего настроения для каждой группы
const moodBySleep = Object.entries(sleepGroups).map(([range, items]) => ({
  range,
  avgMood: items.reduce((sum, e) => sum + e.mood, 0) / items.length
}));

// Вывод: "При 6-8 часах сна ваше настроение на 15% выше"
```

**Шаг 4: Определение трендов**
```javascript
const trend = avgMood > prevAvgMood ? '↑' : avgMood < prevAvgMood ? '↓' : '→';
```

**Шаг 5: Генерация инсайтов**
```javascript
const insights = [
  `Ваше среднее настроение: ${avgMood.toFixed(1)}/10 ${trend}`,
  `Оптимальный сон для вас: ${optimalSleepRange}`,
  `Наиболее продуктивное время: ${peakProductivityTime}`
];
```

### Трудности и решения

**1. Бот засыпал на Render Free Tier**
- **Проблема**: Render усыпляет бесплатные сервисы после 15 мин неактивности
- **Решение**: Переключение с long polling на webhook режим + health check endpoint

**2. IndexedDB сложный API**
- **Проблема**: Низкоуровневое API IndexedDB неудобно
- **Решение**: Использование Dexie.js обертки + создание хелперов

**3. Анимации тормозили на Android**
- **Проблема**: 60 FPS не достигался на старых устройствах
- **Решение**: CSS transforms вместо layout properties, GPU acceleration

**4. Большой размер bundle**
- **Проблема**: Изначально bundle был ~1.5 MB
- **Решение**: Code splitting, tree shaking, minification → 672 KB

**5. Telegram theme colors конфликтовали**
- **Проблема**: Автоцвета Telegram ломали дизайн
- **Решение**: Отключение и использование собственной темной палитры

### Найденные и исправленные проблемы

**1. Иконки навигации исчезали**
- **Проблема**: При активном табе иконки становились невидимыми
- **Причина**: `bg-clip-text text-transparent` применялся к иконкам
- **Решение**: Убран gradient text с иконок, добавлен `drop-shadow`

**2. Просроченные задачи не выделялись**
- **Проблема**: Не было визуальной индикации просрочки
- **Решение**: Добавлена красная граница `border-l-4 border-red-500`

**3. Календарь не отображал записи**
- **Проблема**: Неправильная конвертация дат в ISO формат
- **Решение**: Использование `date.toISOString().split('T')[0]`

**4. AI лимит не сбрасывался**
- **Проблема**: Счетчик запросов не обнулялся в новый день
- **Решение**: Проверка даты при каждом запросе, сброс если день изменился

**5. Анимации лагали на слабых Android**
- **Проблема**: FPS падал до 30 на старых устройствах
- **Решение**: Использование `transform` вместо `top/left`, `will-change`

**6. Модалки не закрывались на Escape**
- **Проблема**: Отсутствовал обработчик клавиатуры
- **Решение**: Добавлен `useEffect` с `addEventListener('keydown')`
